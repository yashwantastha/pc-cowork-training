The one-sentence pitch

You're moving from doing the work with an AI to delegating the work to it and reviewing the result, like managing a capable new hire, not typing into a search box. That's the whole reason Cowork feels different from Gemini, and it's the mental model everything else hangs off.

This is a basic, generic first workshop, deliberately not split by job family. Role-specific use cases come in follow-up sessions built live from today's Wish List answers.

R·T·F·C·G prompt framework

Role · Task · Format · Context · Guardrails. The single habit that most improves output.

Guardrails example: "Don't invent policies, flag gaps, no employee names." New line to remember: guardrails also set tone, e.g. "keep it empathetic and professional, not robotic or legally aggressive," which matters a lot for ER/TA writing.

Token refresher if asked: ~1 syllable per token. A dense page ≈ 500–600 tokens. A 20-page policy ≈ 10–12k tokens.

Connectors, Slack, Projects vs Gems vs Cowork folder
The rule that beats every feature: a connector only shows what you could already see yourself. Default to read-only until trusted.

Claude in Slack (renaming to "Claude Tag" from early Aug 2026) = Claude as an app you message. The Slack connector = Claude reading your channels/DMs as context. Two different things.

Gemini Gems: lightweight, links a live Drive file. Claude Projects: persistent workspace, upload a corpus once + standing instructions, every chat in it starts with that context. Update by adding/removing files, no rebuild needed. Guidance: 2–3 Projects total, consolidate before creating a new one.

Project vs Cowork+folder, the one people mix up: Project = reference library for Q&A, doesn't touch your file system. Cowork+folder = a real folder it can read and write to, for producing/editing an actual file. "Explain from docs we have" → Project. "Go do something, hand me a file" → Cowork.

"Important folder" rule: one dedicated folder per task, never HR/payroll/personnel drives, same idea as a contractor key to one room.

Phase 3 · Agentic + scheduled tasks

Loop: create → revise → verify. Point at one folder, describe the end state (R·T·F·C·G), keep "Ask before acting" on, review the plan before Approve, check output against source.

Scheduled tasks: automate the drafting, never the deciding or sending. Example: Monday 8am new-joiner digest, draft only, human reviews.

Plugins: don't assume a ready-made "HR plugin" exists, check the Plugins panel. Today's recipes work with plain Cowork regardless.

Try It Live — what attendees actually do
  1. Connect Claude: Slack, Gmail, Drive, Calendar. Connect Gemini: Google Workspace toggle. (Auth issue = almost always an org toggle, not user error, flag a facilitator.)
  2. Run the same prompt in both: "Do I have any unread emails that look urgent, and what's on my calendar for the rest of today?" Compare speed / trust / whether it paused to ask.
  3. Claude-only extra: "Summarize what's happened in #[channel] over the last day", proves Gemini's Slack gap concretely.
  4. Project demo (I build this live first): ask a leave-policy question with no Project (generic/guessing answer) → create "Practice Policy Project" with the fictional sample leave policy uploaded → ask same question → compare specificity + citation. This is the exercise that makes Projects click.
  5. Wish List: 1 min, everyone jots one task they wish AI could do. I reality-check 1–2 out loud live ("buildable today" vs "great follow-up idea").
3 HR Recipes (I demo Recipe 1 live)

1. 30-60-90 Onboarding Checklist (People Ops, Cowork) — JD + onboarding policy in one folder → table with Source column per row. Verify: spot-check 3 rows against policy; no source = hallucination candidate.

2. Offer Letter / JD Drafting (Total Rewards, Chat or Cowork) — template + placeholders like [CANDIDATE NAME], never real names. Comp figures always human-validated against the source band before anything sends.

3. Policy Q&A + Comms (any function, Claude Project) — upload current policies, standing instruction "answer only from these docs, cite section, say not-covered if absent."

All practice uses sample/anonymized data only: "Candidate A", fake dates, placeholder bands. Skill transfers, risk doesn't.
Governance bright lines (don't rush this tab)
No autonomous hiring/termination decisions. AI can prepare and organize; a human always decides. This is a high-risk/prohibited pattern under the EU AI Act.
No Special Category Data ever (GDPR Art. 9): health, disability, ethnicity, religion, sexual orientation, union membership, biometric data, plus salaries tied to named individuals, IDs, disciplinary records.

Ops hygiene to reinforce: Ask-before-acting mode, isolated "playground" folders, execution auditing (read the plan before Approve), prompt-injection awareness (a hidden instruction can live inside an uploaded CV/PDF).

Accounts: org-approved enterprise account only, prompts aren't used for training on enterprise terms. Never a personal consumer account for DH data.

Settings tab quick reference
  • Usage → daily spend by product. Only Cowork usage shows; Gemini never appears (expected, it's free).
  • Capabilities → Search/reference chats, Generate memory, Connector search: leave all On. Location metadata: personal choice.
  • Cowork settings → Trusted folders (skip the confirm dialog, never trust an HR/payroll folder) + Global instructions (a one-time briefing applied to every task).
  • Desktop app (device-level, separate from account settings) → General (startup, menu bar, double-tap-Option quick access) and Extensions (Control Chrome, Read/Send iMessages — different from Connectors, only enable deliberately).

FAQ one-liners I might get asked
  • Shared or per-person credit? Per person.
  • Does reading burn credit? Barely, cents. Multi-step tasks all day is what adds up.
  • Confident answer, still check it? Yes, always, every time.
  • Accidentally pasted something sensitive? Stop the task, flag to workshop lead/IT immediately, don't try to fix it by pasting more.
  • Need Claude Code? Almost certainly not.
  • Share a Project with a teammate? Not directly, personal to the account; each person sets up their own from the same source files.
  • What's next after today? Shorter PLT leadership follow-up on department use cases, then possibly a P&C hackathon. Dates not locked.
